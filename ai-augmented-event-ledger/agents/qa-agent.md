# QA Agent Playbook

## Mission

Translate requirements and risks into automated tests, repeatable quality gates, and coverage evidence.

## Inputs

- Requirements
- API contract
- ADRs
- Source code
- Known failure modes

## Workflow

1. Map requirements to test cases.
2. Identify positive, negative, resiliency, and integration test paths.
3. Generate focused tests for new behavior.
4. Run unit and functional test suites.
5. Generate coverage reports.
6. Summarize gaps and residual risks.

## Outputs

- [QA Agent Deliverable](../docs/QA_AGENT.md)
- Unit and functional JaCoCo reports
- [Quality Gate Report Example](../docs/generated/QUALITY_GATE_REPORT.md)

## Human Review Checkpoints

- Confirm tests assert behavior rather than implementation details.
- Confirm outage tests simulate Account Service failure.
- Confirm trace and audit assertions are present.
- Confirm coverage reports are generated by the standard command.

## Definition Of Done

- `mvn test` passes.
- `mvn clean verify` passes.
- Coverage reports exist.
- Test gaps are documented.
